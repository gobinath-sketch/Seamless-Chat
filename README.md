Chat Application

A real time communication platform engineered using an event driven architecture to deliver low latency messaging, scalable connection handling, and seamless interaction across distributed systems.

Overview

This system implements a persistent communication model where clients maintain continuous connections with the server, enabling instantaneous message exchange. The architecture integrates frontend, transport, and backend layers to ensure reliability, scalability, and maintainability.

System Architecture
Layer	Responsibility	Technology
Client Layer	User interface and interaction management	React or modern web stack
Transport Layer	Real time bidirectional communication	WebSocket or Socket.IO
Application Server	Event handling and business logic	Node.js with Express
Data Layer	Data persistence and retrieval	MongoDB or relational systems

Step	Process
1	User submits a message from client
2	Message is transmitted through transport layer
3	Server processes incoming event
4	Target recipient or group is identified
5	Message is routed to appropriate connections

Real Time Synchronization Workflow
Step	Process
1	Server monitors connection lifecycle events
2	User presence state is updated dynamically
3	Active sessions are synchronized across clients
4	Activity signals such as typing are propagated
5	Interface reflects real time system state

Setup and Execution
Repository Initialization

Clone the repository and move into the project directory

git clone https://github.com/gobinath-sketch/chat-appication.git
cd chat-appication
Backend Setup

Navigate to the server directory and install dependencies

cd server
npm install

Start the backend service

npm start
Frontend Setup

Open a new terminal and navigate to the client directory

cd client
npm install

Start the frontend application

npm start
